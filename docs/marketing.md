# Marketing and SEO

## Positioning

OpenClaw Manager is the control plane for operators who run more than one raw OpenClaw project.

It does not try to replace OpenClaw's native single-instance UI.
It adds the missing fleet layer:

- register projects
- inspect live health
- run lifecycle actions
- push repeatable bulk operations
- jump straight into each instance's own Control UI

## Ideal Users

- teams running multiple OpenClaw gateways for production, staging, and experiments
- operators who need one dashboard for many OpenClaw projects
- builders who want bulk hooks, skills, memory, and config actions without inventing a second config system

## Core Messaging

### One-line pitch

One control plane for many raw OpenClaw projects.

### Short description

OpenClaw Manager gives you a fleet dashboard for raw OpenClaw gateways: project inventory, health probes, lifecycle actions, bulk operations, and deep links into each project's native Control UI.

### Differentiators

- built for raw OpenClaw projects, not channel-specific instance types
- keeps single-project depth inside OpenClaw itself
- uses live runtime truth instead of stale database-only status
- supports mixed project layouts, lifecycle commands, and auth inheritance
